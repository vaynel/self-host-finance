## 1. 프로젝트 개요

### 1.1 서비스 소개
**FinFlow**는 개인의 재무 상태를 한눈에 파악하고 관리할 수 있는 웹 기반 개인 재무 관리 플랫폼입니다.

### 1.2 프로젝트 목표
- 사용자의 수입/지출을 체계적으로 기록하고 분석
- 복수 계좌(은행/증권)를 통합 관리
- 투자 포트폴리오 현황 및 수익률 시각화
- 직관적인 리포트를 통한 재무 인사이트 제공
- 데이터 업로드를 통한 일괄 거래 내역 등록

### 1.3 대상 사용자
- 개인 재무 관리가 필요한 일반 사용자
- 투자 포트폴리오를 함께 관리하고 싶은 개인 투자자

## 2. 기술 스택

| 분류 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 | React 18 + TypeScript | SPA 구조 |
| 빌드 도구 | Vite | 빠른 HMR |
| 스타일링 | Tailwind CSS 4 | 유틸리티 기반 |
| UI 컴포넌트 | shadcn/ui + Radix UI | 접근성 준수 |
| 차트 | Recharts | 반응형 차트 |
| 라우팅 | React Router v6 | 클라이언트 사이드 라우팅 |
| 상태 관리 | TanStack Query | 서버 상태 관리 |
| 폼 관리 | React Hook Form + Zod | 유효성 검증 |
| 테스트 | Vitest + Playwright | 단위/E2E 테스트 |

## 4. 페이지 구성 및 기능 명세

### 4.1 로그인 / 회원가입 (`/login`)
| 기능 | 설명 |
|------|------|
| 이메일 로그인 | 이메일 + 비밀번호 기반 인증 |
| 회원가입 | 이름, 이메일, 비밀번호 입력 |
| 비밀번호 재설정 | 이메일 인증을 통한 비밀번호 초기화 |
| 토큰 관리 | JWT Access/Refresh Token 방식 |

### 4.2 대시보드 (`/`)
| 기능 | 설명 |
|------|------|
| 총 자산 현황 | 전체 계좌 잔액 합산 표시 |
| 이번 달 수입/지출 | 현월 기준 수입·지출 합계 |
| 최근 거래 내역 | 최신 5건 거래 표시 |
| 월별 수입/지출 차트 | 최근 6개월 막대그래프 |
| 카테고리별 지출 | 원형 차트 시각화 |

### 4.3 거래 내역 (`/transactions`)
| 기능 | 설명 |
|------|------|
| 거래 목록 | 전체 거래 내역 테이블 형태 표시 |
| 필터링 | 거래 유형, 카테고리, 계좌별 필터 |
| 검색 | 거래 내용 텍스트 검색 |
| 거래 추가 | 수동 거래 등록 |
| 거래 삭제 | 개별 거래 삭제 |

### 4.4 계좌 관리 (`/accounts`)
| 기능 | 설명 |
|------|------|
| 계좌 목록 | 은행/증권 계좌 카드형 표시 (카드 제외) |
| 잔액 흐름 차트 | 계좌 클릭 시 라인 차트로 잔액 추이 표시 |
| 거래 내역 연동 | 계좌별 관련 거래 내역 조회 |
| 계좌 추가/삭제 | 계좌 정보 관리 |

### 4.5 투자 관리 (`/investments`)
| 기능 | 설명 |
|------|------|
| 보유 종목 현황 | 종목별 보유수량, 평균단가, 현재가, 수익률 표시 |
| 종목 시세 차트 | 최근 30일 일별 종가 차트 |
| 매매 내역 | 매수/매도 거래 이력 |
| 포트폴리오 비중 | 원형 차트로 비중 시각화 |

### 4.6 리포트 (`/reports`)
| 기능 | 설명 |
|------|------|
| 월별 요약 | 월별 수입, 지출, 저축률 요약 |
| 카테고리 분석 | 카테고리별 지출 비중 분석 |
| 추세 분석 | 월별 수입/지출 추세 라인 차트 |

### 4.7 데이터 업로드 (`/upload`)
| 기능 | 설명 |
|------|------|
| 파일 업로드 | CSV/XLSX 파일 드래그&드롭 업로드 |
| 데이터 매핑 | 파일 컬럼과 시스템 필드 매핑 |
| 미리보기 | 업로드 전 데이터 확인 |
| 결과 리포트 | 처리 결과 (성공/실패/건너뜀) 표시 |

### 4.8 설정 (`/settings`)
| 기능 | 설명 |
|------|------|
| 프로필 관리 | 사용자 이름, 이메일 수정 |
| 통화 설정 | 기본 통화 변경 |
| 알림 설정 | 알림 수신 여부 설정 |

## 5. 데이터 모델

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| email | string | 이메일 (unique) |
| name | string | 사용자 이름 |
| password | string | 해시된 비밀번호 |
| createdAt | timestamp | 가입일 |

### 5.2 Transaction (거래)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| userId | UUID | 사용자 FK |
| date | date | 거래 날짜 |
| description | string | 거래 내용 |
| amount | number | 금액 (지출은 음수) |
| type | enum | `income` / `expense` / `transfer` |
| category | string | 카테고리 |
| account | string | 계좌명 |
| memo | string | 메모 (선택) |

### 5.3 Account (계좌)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| userId | UUID | 사용자 FK |
| name | string | 계좌 별칭 |
| type | enum | `bank` / `investment` |
| balance | number | 현재 잔액 |
| institution | string | 금융기관명 |
| lastSync | timestamp | 마지막 동기화 시각 |

### 5.4 InvestmentTrade (투자 매매)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| userId | UUID | 사용자 FK |
| ticker | string | 종목 코드 |
| name | string | 종목명 |
| type | enum | `stock` / `etf` |
| action | enum | `buy` / `sell` |
| date | date | 거래 날짜 |
| shares | number | 수량 |
| price | number | 단가 |
| fee | number | 수수료 (선택) |

## 7. UI/UX 설계 원칙

### 7.1 디자인 시스템
- **색상 팔레트**: HSL 기반 시맨틱 토큰 사용
  - Primary: `hsl(172, 66%, 40%)` — 민트/틸 계열
  - Background: `hsl(220, 20%, 97%)` — 밝은 회색
  - Sidebar: `hsl(220, 25%, 10%)` — 다크 네이비
- **타이포그래피**: Inter (본문) + JetBrains Mono (숫자/코드)
- **모서리 반경**: `0.75rem` 기본
- **다크모드**: CSS 변수 기반 자동 전환 지원

### 7.2 레이아웃
- 좌측 고정 사이드바 (접힘 지원)
- 상단 헤더 (페이지 제목 + 알림)
- 반응형 디자인 (모바일 사이드바 시트 전환)

### 7.3 컴포넌트 규칙
- shadcn/ui 기반 재사용 가능한 컴포넌트 활용
- 모든 색상은 CSS 변수 시맨틱 토큰으로 참조
- 접근성 (a11y) 기본 준수 (ARIA, 키보드 네비게이션)

## 9. 보안 요구사항

- 비밀번호는 bcrypt 해싱 후 저장
- JWT 토큰 만료 시간: Access 1시간, Refresh 7일
- HTTPS 필수 (TLS 1.3)
- SQL Injection / XSS / CSRF 방어
- Row Level Security (RLS) 적용
- 사용자별 데이터 격리 보장
